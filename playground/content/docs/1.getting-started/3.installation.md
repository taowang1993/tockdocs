## `create-tockdocs` CLI

::steps

### Create your docs directory

Use the `create-tockdocs` CLI to create a new TockDocs project:

```bash [Terminal]
npx create-tockdocs my-docs
```

You can choose between two templates:

- `default`: Basic TockDocs setup for single-language documentation
- `i18n`: Includes internationalization support for multi-language documentation

```bash [Terminal]
# Create with i18n template
npx create-tockdocs my-docs -t i18n
```

We recommend using the `npm` package manager.

### Start your docs server in development

Move to your docs directory and start your docs server in development mode:

```bash [Terminal]
cd my-docs
npm run dev
```

A local preview of your documentation will be available at <http://localhost:3000>

### Write your documentation

Head over the [Markdown Syntax](/essentials/markdown-syntax) section to learn how to write your documentation.
::

## Layer Integration

TockDocs v4 uses a **Nuxt layer-based approach**, you can extend the TockDocs layer directly in your `nuxt.config.ts` with `extends: ['tockdocs']`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['tockdocs'],
})
```
