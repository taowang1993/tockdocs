## **Migrating from TockDocs v3 to v4**

TockDocs v4 introduces a new **layer-based approach** that leverages the official Nuxt CLI instead of the custom TockDocs CLI. While your existing content and configuration remain compatible, you'll need to update your commands and project setup.

### **⚠️ Breaking Changes**

The main breaking changes are related to CLI commands:

| v3                          | v4                              |
| --------------------------- | ------------------------------- |
| `npx tockdocs init my-docs` | `npx create-tockdocs my-docs`   |
| `tockdocs dev`              | `nuxt dev --extends tockdocs`   |
| `tockdocs build`            | `nuxt build --extends tockdocs` |

::tip
Your existing Markdown content and MDC syntax will work without changes. The migration primarily involves updating your development and build workflow.
::

## **Migrating to TockDocs**

Already using a Markdown-based solution for your documentation? Whether it’s **TockDocs v1**, the **Nuxt UI docs template**, or another static site setup, migrating to TockDocs is simple and straightforward.

TockDocs offers a clean and maintainable solution with a single dependency: the TockDocs library itself. There’s no need to manage multiple dependencies. With everything built-in and maintained together, keeping your documentation up to date is easier than ever.

To migrate, just move your existing Markdown files into the `content/` directory of the TockDocs starter.

From there, you have two scenarios:

- **If your current docs already use Nuxt Content and the MDC syntax**, make sure the components used in your content exist in Nuxt UI. If any components are missing, you can easily create your own custom ones.
- **If you're using standard Markdown**, you can copy your files as is. Then, enhance your documentation progressively using the built-in components provided by Nuxt UI.

Once your content has been moved to the `content/` folder, you can customize your app by following the [configuration documentation](https://tockdocs.dev/concepts/configuration).

TockDocs is designed to focus on writing content, so if you're already using Markdown, you can easily switch to it.
