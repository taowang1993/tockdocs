## CLI `create-tockdocs`

::steps

### Créez votre dossier de documentation

Utilisez le CLI `create-tockdocs` pour créer un nouveau projet TockDocs :

```bash [Terminal]
npx create-tockdocs my-docs
```

Vous pouvez choisir entre deux templates:

- `default` : Configuration TockDocs de base pour une documentation monolingue
- `i18n` : Inclut le support d'internationalisation pour une documentation multilingue

```bash [Terminal]
# Créer avec le modèle i18n
npx create-tockdocs my-docs -t i18n
```

Nous recommandons d'utiliser le gestionnaire de paquets `npm`.

### Démarrez votre serveur de documentation en mode développement

Déplacez-vous dans votre dossier de documentation et démarrez votre serveur:

```bash [Terminal]
cd my-docs
npm run dev
```

Un aperçu de votre documentation sera disponible sur <http://127.0.0.1:4987>

### Rédigez votre documentation

Rendez-vous dans la section [Syntaxe Markdown](/fr/essentials/markdown-syntax) pour apprendre à rédiger votre documentation.
::

## Intégration du layer TockDocs

TockDocs v4 utilise une **approche basée sur les layers Nuxt**, vous pouvez étendre le layer TockDocs directement dans votre `nuxt.config.ts` avec `extends: ['tockdocs']` :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['tockdocs'],
})
```
